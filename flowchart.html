🎭 How Playwright Works

From config file → browser actions → pass or fail

⚙️ playwright.config.js
Sets browser · baseURL · timeout · headless · slowMo
SETTINGS
📋 login.spec.js
Defines test groups (describe) and individual tests (test)
TEST FILE
🔁 test.beforeEach runs
Creates LoginPage · calls goto() · opens the browser
📄 LoginPage.js (Page Object)
Finds elements by #id · .class · role · label
LOCATORS & ACTIONS
🌐 Real Browser Opens (Chromium)
Navigates to URL · renders the page
🖱️ Actions Execute (one by one)
fill username → fill password → click Submit
✅ expect() Assertions Run
Check URL · text · visibility · value
✅ PASS
Test marked green
Moves to next test
❌ FAIL
Screenshot saved
Error shown in terminal

⚙️ Config Options

  • Browser chromium / firefox / webkit
  • headless true = hidden, false = visible
  • slowMo delay between actions (ms)
  • timeout max time per test
  • baseURL root URL of your site
  • retries retry count on failure

📄 Locator Types

  • By ID #username
  • By class .submit-btn
  • By role getByRole('button')
  • By label getByLabel('Email')
  • By text getByText('Login')
  • By placeholder getByPlaceholder('...')

✅ Assertion Types

  • Visible toBeVisible()
  • Hidden toBeHidden()
  • URL check toHaveURL(/pattern/)
  • Text match toHaveText('...')
  • Partial text toContainText('...')
  • Input value toHaveValue('')
